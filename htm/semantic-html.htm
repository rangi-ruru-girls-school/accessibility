Semantic HTML

Semantic elements are elements with a meaning. A semantic element clearly describes its meaning to both the browser and the developer. Make your HTML code as semantic as possible, so that the code is easy to understand for visitors and screen readers.

Examples of non-semantic elements: <div> and <span> - Tells nothing about its content.

Examples of semantic elements: <header>, <nav>, and <main> - Clearly defines its content.

Some things you can do to make your HTML more semantic:

  • Headings: search engines use headings to index the structure and content of your web pages.
  • Alternative text: the alt attribute provides text for an image incase it's broken or the user can't view it (because of slow connection, an error in the src attribute, or if the user uses a screen reader.
  • Language: screen readers and search engines both use the lang attribute to know the language the page is in.
  • Link title: giving your links a title attribute helps by showing a tooltip when the user hovers over the link.
